1. 메일 전송 시 (smtp 사용)

  outlook, gmail 에서 정크 메일로 차단됨.
  jinhakapply 로 메일 발송 하려면 정크 메일로 가면 안됨.
  해결 방법?
  A) 진학사 서버에서 전송하면 문제 없음 (나중에 해결할 문제임)

2. 에디터의 경우 
  - 다른 에디터를 사용해보고 싶은데 javascript에서 이식하는 방법으로 가능한건지?
  - 표 삽입, 이미지 삽입 연관하여 현재 쓰고 있는 컨트롤의 내부 소스를 수정할 수 있는지?
  * 표 삽입 크기 변경 가능하도록, 표 안에 텍스트 입력 시 마음대로 바뀌는 부분 수정필요
  * 이미지 삽입 동작 안됨
  A)  에디터의 경우에는 충분히 많고, 그에 해당하는 CSS 속성을 바꿔줌으로 위사항은 해결될 것임
      에디터에 맞게 해줘야하기 때문에 추후에 구현할 예정
  
3. 페이지 확대, 축소 시 표의 형태가 깨지게 되는 현상 수정 필요
  A)  페이지 확대, 축소 시 깨지는 것은 일반적인 현상이며, 대부분의 경우 페이지 확대 축소를 안하고
      굳이 만들어두고 다른 사람이 안쓸수도 있기에 만들 필요 없음. (요청사항을 명확히 파악하는 것이 중요함)

4. css 중첩해서 사용하는 경우
  css 우선순위를 바꾸고 싶을 때?
  A) CSS 우선순위

    2) !important를 입힌 것
    3) css 참조 위에서 참조하냐 아래서 참조하냐의 차이가 있음


<리뷰 사항 정리>
1. 전체 flow
  - 회원가입은 필요 없음
  - 로그인 내부망에서 접속할 땐 dev 서버에서 쓰면 되고, ip 기반으로 접속할 수 있음. 외부망은 생각해야할게 많음.
  - 사용자
    : 운영부 (요청, 응답)
    : 개발부 (처리, 답변) - 4명 정해져있음

  - 동작절차
    1) 운영자: 내가 요청한 내용만 보이면 됨.
    2) 개발자: 
      * 맨처음 화면엔 요청한 내용만 와야한다.
      * 운영자가올린 요청을 조회하는 시점에 "처리중"으로 바뀌고 
      * "처리중"->"완료" 바꾸면 끝.
    3) 운영자: "완료"된 글을 보고 확인하면 해당 글은 <완료목록>으로 옮겨지게 됨
    ※ 운영자가 요청했으나 1일이상 넘어가면 "지연"으로 바뀔 것. 
    ※ "불가"인 경우에는 기타사유를 입력할 수 있어야함.

2. 운영자
  - 작성할 수 있는 글의 종류는 (1) 오류, (2) 데이터, (3) 기타 3가지이다.
  - 핵심은 운영자가 작성할 때 편하게 작성하는 것이다. 그리고 요청의 속성에 맞게 제작해야한다.
  - 1) 오류
    : 오류 내용
    : 현상(스크린샷), 재현방법 - 관리자, 프론트, 상황판 페이지
  - 2) 데이터
    : 엑셀 파일을 올릴 수 있도록 하고 추후엔 DB에 저장되게 만들기
  - 3) 기타문의
    : 내용을 작성할 수 있어야 함. (서비스 아이디, 서비스 명 넣는것 어떤식으로 진행할것인지 생각해봐야 함.)

    ※삭제란건 없다.
    ※수정은 요청중, 처리중에서 댓글로 작성하게 만들어주고. (재요청)을 씌워서 다시 띄울 것.
      - 댓글에 에디터 입혀줘야함.
      - (재요청)을 씌울 땐 새로 글을 만들어서 최신 글로 띄울 것이냐 아니면 해당 글을 (재요청)을 씌워서 위로 올릴 것이냐.

3. 개발자
  - 오류 => 어떻게 오류가 뜨는지 확인할 수 있도록 링크로 만들 것.
  - 데이터 => 엑셀파일의 데이터 베이스 화
  - 기타 => 천차만별임, 

4. 진행 방안
  - PIMS 상황판에 일단 구현해보기 (JQuery, Javascript, 서버기반 asp.net)
  - 상황판 V2 에 적용
  - 추후 필요하면 EIM에 적용
  

  ※ 추후 외부망에서 사용할 수 있게 하려면 생각해줘야할 것들
  - 로그인 시 필요한 개인정보 처리..
  
  ※ 일단 기존 소스코드 분석해보고 내가 할 수 있는 것과 할 수 없는 것을 구분할 것.
